입력받은 주제/URL: $ARGUMENTS

이 저장소(Tech_Storage)의 새 기술동향 문서를 작성한다. 절차는 CLAUDE.md의 "문서 작성 절차"를 그대로 따른다:

1. 입력이 URL/제품명이면 먼저 정체를 조사해 확인한다 (지레짐작 금지).
2. Agent로 심층 리서치, 별도 Agent로 인용할 링크·통계 fact-check.
3. 구조: 왜 필요한가 → 비유 → 개념 → 실전 사례 → 비교 → 핵심 원칙 → 참고자료.
4. mermaid 다이어그램 1~3개 + "그림 읽는 법" 캡션.
5. 비전문가도 읽을 수 있게 쓰되 깊이는 유지. 불확실한 내용은 ⚠️ 경고 블록으로 명확히 분리.
6. 문서 맨 끝에 `📎 더 많은 기술동향: https://github.com/21-Arbiter/Tech_Storage` 추가.
7. 알맞은 `기술동향/<주제폴더>/YYYY-MM-DD-슬러그.md`에 저장 (없는 주제면 새 폴더+README 생성).
8. 해당 폴더 README.md 목록 + 루트 README.md 목차 둘 다 갱신.
9. git commit — **push는 사용자에게 반드시 먼저 확인받는다.**
